Accessibility

I want everyone to be able to read and understand the content of my website.

How accessible is my website?

Plugins suggested used at your own risk. I am not responsible for any issues you may have from additional plugins.

Images on my site

  • All images that add more information and are important on some pages have an alt included. This is particularly important on the quiz and degree programme page. But not so much for the logo as it is purely decorative so this needs changing in the future. It will allow visually impaired users to use screen reader plugins to access a description of my photo and add context to the module descriptions and questions. It also allows image to be converted to braille or large print if it is needed in this format.

Text Content

  • All my text content provided in the main section of my pages can be read out by "read aloud" features such as what can be found on microsoft edge.
  • Font size of text on my pages can be altered by the visitor using zoom magnification in browser settings for both mobile and desktop sites.
  • If the visitor is not happy with the text colour as this has been identified as a problem in testing, then they could use HighContrast which can be found on the chrome web store.
  • Text content on all pages follows a logical and linear layout for both mobile and desktop.
  • Although not a problem, changing font style may help accessibility and this is something for me to look at as an update to the site.

Navigation

  • All my navigation links are accessible through the use of only a keyboard. The tab being focused on is very clear when using a keybaord
  • The structure of my navigation is logical and linear, allowing the user to quickly get to another page of my website.
  • All buttons have context.

My content is understandable

  • It helps user avoid and correct mistakes such as validation on the form I provide on the contact page.
  • All content is in logical order and it operates in a predictable way such as the quiz page. (Answers follow the question)

How I tested the accessibility of my website?

  • I used the Google Chrome Lighthouse to quickly audit my pages for both the mobile and desktop views. Overall results were very good with the majority of pages receiving a score of 90+.
  • I manually tested some features such as using the microsoft edge "read aloud" tool and using my keyboard to navigate the site.

If you have a problem with the accessibility of this site and would like to send me feedback. Please use the form I have provided on the Contact Page.
